# ./dihedrals_analysis.py #

# Overview #

This script computes backbone and side-chain torsion angle distributions from MD trajectories using MDAnalysis. It:

 - Defines residue-wise torsion (φ, ξ, χ) angles via custom atom selections
 - Computes dihedral angles across trajectory frames
 - Builds a 3D kernel density estimate (KDE) of angle distributions
 - Saves data and generates 3D density visualizations

# Usage #

`python script.py <reference.pdb> <trajectory.xtc>`

Arguments

`<reference.pdb> — reference, NAMED_structure. Atom selection relies on the resids and atom names generated by urethane_top_MD`

`<trajectory.pdb> — population of a cluster in pdb trajectory`

# Output #

1. Raw torsion data
`<traj>.3dkde.data.npy`

Shape:

(n_frames × residues, 3)

Columns:

 - φ (phi)
 - ξ (xi)
 - χ (chi)

2. KDE density

`<traj>.3dkde.npz`

Contains:

 - PDF → 3D density grid
 - coordinates → grid coordinates
 - grid_limits, resolution
 - labels → angle names

3. Visualization
`<traj>.3dkde.png`

3D kernel density plot of torsion angle space
Shows conformational preferences across trajectories

**Notes**

- Torsions are defined per residue using custom atom selections
- Kernel width is fixed. Geussing the width takes long time.
- Requires consistent atom naming across system
- KDE computed on a fixed grid: [-180, 180]^3
- Heavy computation due to per-residue dihedral evaluation
